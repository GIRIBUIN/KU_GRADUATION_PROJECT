# 데이터 모델

## 1. 개요

모든 작업은 하나의 `Task` 모델로 통합한다. 개인 task와 e-campus task는 같은 구조에서 관리하고, e-campus task에만 동기화용 메타데이터를 추가한다.

## 2. Task

| 필드 | 타입 | 설명 |
| --- | --- | --- |
| `id` | String | 앱 내부 task 고유 ID |
| `origin` | Enum | `personal`, `ecampus` |
| `status` | Enum | `active`, `completed`, `deleted`, `excluded` |
| `title` | String | 사용자가 보는 제목 |
| `dueDate` | DateTime? | 사용자가 보는 마감일 |
| `priority` | Enum? | task 직접 우선순위 |
| `memo` | String? | 메모 |
| `parentTaskId` | String? | parent task ID |
| `createdAt` | DateTime | 생성 시각 |
| `updatedAt` | DateTime | 수정 시각 |
| `completedAt` | DateTime? | 완료 시각 |
| `deletedAt` | DateTime? | 삭제 시각 |
| `tagIds` | List<String> | 연결된 태그 ID 목록 |
| `folderIds` | List<String> | 연결된 폴더 ID 목록 |
| `sortOrder` | Int | 리스트 정렬을 위한 순서 값 |

## 3. e-campus Sync Metadata

e-campus task인 경우에만 사용한다.

| 필드 | 타입 | 설명 |
| --- | --- | --- |
| `sourceKey` | String? | e-campus 항목 식별용 내부 키 |
| `sourceTitle` | String? | e-campus 원본 제목 |
| `sourceDueDate` | DateTime? | e-campus 원본 마감일 |
| `sourceCourse` | String? | 과목명 |
| `sourceType` | EcampusTaskType? | 과제, 온라인강의, 팀프로젝트 등 |
| `lastSyncedAt` | DateTime? | 마지막 동기화 시각 |

`sourceKey`는 e-campus HTML에 그대로 존재하는 필드명이 아니라 앱에서 만든 동기화 식별 키이다.
현재 확인한 체크리스트 HTML 기준으로는 `goLecture(...)` 인자, `kj_*` hidden input의 `value`, `gubun_*` hidden input의 `value`를 조합해 만든다.

## 4. Task Status

| 값 | 설명 |
| --- | --- |
| `active` | 진행 중 |
| `completed` | 완료됨 |
| `deleted` | 삭제됨, 복구 가능 |
| `excluded` | e-campus에서 발견됐지만 가져오지 않음 |

`excluded`는 e-campus task에만 사용할 수 있다.

## 5. Sub Task

현재 구현에서 sub task는 `Task`의 `parentTaskId`가 아니라 별도 `SubTasks` 테이블로 표현한다.

| 필드 | 타입 | 설명 |
| --- | --- | --- |
| `id` | String | sub task ID |
| `taskId` | String | parent task ID |
| `title` | String | sub task 제목 |
| `isDone` | Boolean | 완료 여부 |
| `createdAt` | DateTime | 생성 시각 |
| `updatedAt` | DateTime | 수정 시각 |

`Task.parentTaskId` 필드는 모델과 DB에 남아 있지만, 현재 화면/저장 흐름의 sub task 기능은 `SubTask.taskId`를 사용한다.

## 6. Tag

| 필드 | 타입 | 설명 |
| --- | --- | --- |
| `id` | String | 태그 ID |
| `name` | String | 태그 이름 |
| `color` | String | 표시 색상 |
| `createdAt` | DateTime | 생성 시각 |
| `updatedAt` | DateTime | 수정 시각 |

Task와 Tag는 many-to-many 관계이다.

태그 기본 우선순위 필드는 현재 구현되어 있지 않다. 태그 숨김, 태그-폴더 배치, 태그 정렬 순서는 `AppSettings`에 저장한다.

## 7. Folder

| 필드 | 타입 | 설명 |
| --- | --- | --- |
| `id` | String | 폴더 ID |
| `name` | String | 폴더 이름 |
| `color` | String? | 폴더 색상 |
| `icon` | String? | 폴더 아이콘 |
| `parentFolderId` | String? | 상위 폴더 ID |
| `sortOrder` | Int | 표시 순서 |
| `createdAt` | DateTime | 생성 시각 |
| `updatedAt` | DateTime | 수정 시각 |

Task와 Folder는 many-to-many 관계이다.

## 8. Notification

task 하나에는 알림 설정 하나만 둔다.

| 필드 | 타입 | 설명 |
| --- | --- | --- |
| `id` | String | 알림 ID |
| `taskId` | String | 연결된 task ID |
| `enabled` | Boolean | 알림 사용 여부 |
| `daysBeforeDue` | Int | 마감 몇 분 전 알림인지. 필드명과 달리 현재 구현은 분 단위 |
| `notifyTime` | String | 알림 시간 또는 방식. 현재 기본 설정은 `relative` |
| `scheduledAt` | DateTime? | 실제 예약된 알림 시각 |

## 9. 우선순위 계산

```text
if task.priority exists:
    use task.priority
else:
    no priority
```

태그 기본 우선순위와 앱 기본 우선순위 계산은 현재 구현되어 있지 않다. 신규 e-campus task 생성 시에는 기본값으로 `medium`을 저장한다.

## 10. App Settings

설정은 `AppSettings` key-value 테이블에 문자열로 저장한다.

| 값 | 설명 |
| --- | --- |
| `defaultNotificationEnabled` | 기본 알림 사용 여부 |
| `defaultNotificationDays` | 기본 알림 오프셋. 현재 분 단위 |
| `defaultNotificationTime` | 기본 알림 방식. 현재 `relative` |
| `urgentDueDays` | 홈의 마감 임박 기준 일수 |
| `homeSelectedTagId` | 홈 태그 섹션에서 선택된 태그 |
| `hiddenTagIds` | 숨긴 태그 ID 목록 |
| `hiddenFolderIds` | 숨긴 폴더 ID 목록 |
| `tagFolderIds` | 태그 ID와 폴더 ID 매핑 |
| `tagSortOrders` | 태그 표시 순서 |
| `ecampusFolderId` | 자동 생성/연결된 e-campus 폴더 ID |

`autoSyncEnabled`, `saveEcampusAccount` 값도 모델에 남아 있지만, 현재 UI와 동기화 흐름에서는 자동 동기화와 계정 저장 기능을 제공하지 않는다.

## 11. 삭제 기록 유지

### 개인 task

- `status = deleted`
- `deletedAt` 기준 7일 후 완전 삭제 가능

### e-campus task

- `status = deleted` 또는 `excluded`
- `sourceKey`, `sourceDueDate` 유지
- `sourceDueDate` 이후 정리 가능
- `sourceDueDate`가 없으면 `deletedAt` 또는 `updatedAt` 기준 7일 후 정리 가능
