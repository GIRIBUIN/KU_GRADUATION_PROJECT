# 데이터 수집 및 가공 설계

## 1. 수집 대상

e-campus 체크리스트에서 다음 유형의 ToDo를 수집한다.

- 온라인 강의
- 과제
- 팀프로젝트 과제
- 기타 e-campus 체크리스트에 표시되는 학업 작업

학교 행사나 축제 일정 같은 일반 캘린더 이벤트는 수집 대상이 아니다.

## 2. 수집 데이터

e-campus HTML에서 가능한 범위 내에서 다음 값을 추출한다.

| 값 | 설명 |
| --- | --- |
| `sourceKey` | e-campus 항목 식별용 내부 키 |
| `title` | 원본 제목 |
| `course` | 과목명 |
| `dueDate` | 마감일 |
| `dDay` | D-day 표시 |
| `type` | 과제, 온라인강의, 팀프로젝트 등 |

`sourceKey`는 HTML에 있는 단일 필드명이 아니다.
현재 확인한 체크리스트 HTML 기준으로는 `goLecture(...)` 인자, `kj_*` hidden input의 `value`, `gubun_*` hidden input의 `value`를 파싱한 뒤 조합해 만든다.

## 3. Task 변환 규칙

수집한 e-campus 항목은 앱 내부 Task로 변환한다.

| e-campus 값 | Task 필드 |
| --- | --- |
| 조합한 식별 키 | `sourceKey` |
| 원본 제목 | `title`, `sourceTitle` |
| 원본 마감일 | `dueDate`, `sourceDueDate` |
| 과목명 | `sourceCourse` |
| 유형 | `sourceType` |

가져온 이후의 `title`, `dueDate`, `priority`, `tags`, `memo`, `notification`은 사용자가 자유롭게 수정할 수 있다.

## 4. 신규 항목 판단

```text
if parsed.sourceKey not in local DB:
    신규 항목
else:
    기존 항목
```

`sourceKey`가 다르면 기본적으로 다른 task로 본다.

## 5. 업데이트 후보 판단

```text
if parsed.sourceKey exists in local DB:
    compare sourceTitle, sourceDueDate, sourceCourse, sourceType
    if any source field changed:
        업데이트 후보
```

업데이트 후보는 자동 반영하지 않고 동기화 결과 화면에서 사용자 선택을 받는다.

## 6. 동기화 반영

### 신규 항목 선택 시

- 새 Task 생성
- `origin = ecampus`
- `status = active`
- `sourceKey`, `sourceTitle`, `sourceDueDate`, `sourceCourse`, `sourceType` 저장
- `priority = medium`
- `memo = course`
- 과목명 태그를 자동 생성/연결
- `e-campus` 폴더를 자동 생성/연결

### 업데이트 후보 선택 시

- 기존 Task를 원본 기준으로 업데이트
- `title`, `dueDate`, `memo`, e-campus 메타데이터를 원본 값으로 덮어쓴다.
- 기존 task의 직접 우선순위, sub task, 완료/삭제 시각은 유지한다.
- 과목명 태그와 `e-campus` 폴더가 없으면 추가 연결한다.
- 사용자가 선택하지 않으면 기존 Task 값을 유지한다.

## 7. 가져오지 않을 항목 처리

사용자가 동기화 결과에서 항목을 가져오지 않으면 다음 값을 저장한다.

- `sourceKey`
- `sourceDueDate`
- `status = excluded`

다음 동기화 때 같은 항목은 신규가 아니라 `가져오지 않을 항목`으로 표시한다.

## 8. 삭제된 e-campus 항목 처리

e-campus task를 사용자가 삭제한 경우:

- 목록에서는 숨긴다.
- `status = deleted`
- `sourceKey`와 `sourceDueDate`는 유지한다.
- 다음 동기화 때 같은 항목은 `가져오지 않을 항목`으로 표시한다.

## 9. 정리 정책

- 개인 task 삭제 기록은 7일 뒤 정리 가능하다.
- e-campus 삭제/제외 기록은 `sourceDueDate` 이후 정리 가능하다.
- `sourceDueDate`가 없는 경우 7일 뒤 정리 가능하다.

## 10. 파싱 구현 메모

현재 구현은 `lib/data/services/ecampus_todo_parser.dart`의 `EcampusTodoParser`를 기준으로 동작한다.

구현 기준:

- `div.todo_wrap` 항목을 순회한다.
- `#no_data` 또는 `no_data` 클래스 항목은 건너뛴다.
- 제목(`.todo_title`), 과목명(`.todo_subjt`), sourceKey 구성 필드가 없으면 실패 항목으로 분리한다.
- D-day 또는 날짜 라벨에서 마감일을 계산하며, 시간 정보가 없으면 23:59를 사용한다.
- 파싱 실패 시 앱이 중단되지 않게 하고 오류 항목으로 분리한다.
- 민감정보는 로그에 출력하지 않는다.
