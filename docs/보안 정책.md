# 보안 정책

## 1. 기본 원칙

자체 서버 없이 사용자의 기기에서 task와 계정 정보를 관리한다. e-campus 계정 정보는 사용자 동의가 있는 경우에만 기기 내 보안 저장소에 저장하며, 별도 서버로 전송하거나 저장하지 않는다.

## 2. 계정 정보 저장

- 계정 정보 저장은 사용자 명시 동의 기반이다.
- 최초 로그인 또는 계정 입력 시 저장 여부를 묻는다.
- 설정 화면에서 저장 여부를 변경할 수 있다.
- 설정 화면에서 저장된 계정 정보를 삭제할 수 있다.

문서와 발표에서는 다음과 같이 표현한다.

```text
사용자 동의를 받은 경우, 계정 정보는 기기 내 보안 저장소에 암호화 저장되며 자체 서버에는 저장되지 않는다.
```

## 3. 동기화 세션 관리

- 동기화가 필요한 순간에만 로그인한다.
- 체크리스트 수집이 끝나면 로그아웃한다.
- 세션과 쿠키를 정리한다.
- 장시간 로그인 상태를 유지하지 않는다.

## 4. 자동 동기화

- 자동 동기화는 사용자가 설정에서 켠 경우에만 동작한다.
- 저장된 계정 정보가 있는 경우 앱 실행 시 1회 시도한다.
- 실패 시 사용자에게 안내하고 반복적으로 방해하지 않는다.

## 5. 로컬 데이터 보호

- task 데이터는 기기 내부 저장소에 저장한다.
- 계정 정보는 일반 저장소가 아니라 보안 저장소에 저장한다.
- 민감정보는 로그에 남기지 않는다.
- 디버깅 출력에 아이디, 비밀번호, 세션, 쿠키를 포함하지 않는다.

## 6. e-campus 접근 정책

- e-campus 데이터는 읽기 전용으로만 사용한다.
- 제출, 수정, 삭제 요청을 e-campus에 보내지 않는다.
- 수집 범위는 체크리스트 ToDo로 제한한다.
- 동기화 빈도를 과도하게 높이지 않는다.

## 7. 사용자 제어권

사용자는 다음 항목을 직접 제어할 수 있어야 한다.

- 계정 정보 저장 여부
- 자동 동기화 여부
- 저장된 계정 삭제
- 동기화 결과에서 가져올 항목 선택
- 가져오지 않을 항목 복구
- 삭제된 task 복구

## 8. 보안 리스크와 대응

| 리스크 | 대응 |
| --- | --- |
| 계정 정보 저장 우려 | 사용자 동의, 기기 내 보안 저장소, 삭제 기능 제공 |
| 세션 탈취 우려 | 동기화 후 로그아웃 및 세션/쿠키 정리 |
| 과도한 e-campus 접근 | 수동 동기화 중심, 자동 동기화는 앱 실행 시 1회 |
| 민감정보 로그 노출 | 계정, 쿠키, 세션 로그 출력 금지 |
| HTML 구조 변경 | 파싱 실패 처리와 오류 안내 |
