# 보안 정책

## 1. 기본 원칙

자체 서버 없이 사용자의 기기에서 task와 e-campus 로그인 세션을 관리한다. 현재 구현은 e-campus 아이디와 비밀번호 저장 기능을 제공하지 않으며, WebView 로그인 후 수집한 쿠키 기반 세션으로 체크리스트를 조회한다.

## 2. 계정 정보 저장

- 현재 구현은 아이디/비밀번호를 앱 설정이나 로컬 DB에 저장하지 않는다.
- 로그인은 e-campus WebView에서 사용자가 직접 수행한다.
- 로그인 성공 후 쿠키를 `EcampusSession`으로 읽어 todo 조회 요청에 사용한다.
- 설정 화면에서 세션/쿠키 정리를 실행할 수 있다.

문서와 발표에서는 다음과 같이 표현한다.

```text
아이디와 비밀번호는 앱에 저장하지 않으며, e-campus WebView 로그인 세션으로 todo를 가져온다. 자체 서버에는 계정 정보와 세션 정보를 저장하지 않는다.
```

## 3. 동기화 세션 관리

- 동기화가 필요한 순간에만 로그인한다.
- 로그인 세션이 있으면 재사용하고, 없거나 만료되면 WebView 로그인으로 이동한다.
- 사용자가 설정 화면에서 세션과 쿠키를 정리할 수 있다.
- 로그아웃 요청은 세션/쿠키 정리 동작에서 수행한다.

## 4. 자동 동기화

- 현재 구현에는 앱 실행 시 자동 동기화 UI와 실행 흐름이 없다.
- 동기화는 홈 또는 설정 화면에서 사용자가 직접 실행한다.

## 5. 로컬 데이터 보호

- task 데이터는 기기 내부 저장소에 저장한다.
- 계정 아이디와 비밀번호는 저장하지 않는다.
- 세션/쿠키는 WebView 쿠키 저장소와 앱 메모리의 `EcampusSession`으로만 다룬다.
- 민감정보는 로그에 남기지 않는다.
- 디버깅 출력에 아이디, 비밀번호, 세션, 쿠키를 포함하지 않는다.

## 6. e-campus 접근 정책

- e-campus 데이터는 읽기 전용으로만 사용한다.
- 제출, 수정, 삭제 요청을 e-campus에 보내지 않는다.
- 수집 범위는 체크리스트 ToDo로 제한한다.
- 동기화 빈도를 과도하게 높이지 않는다.

## 7. 사용자 제어권

사용자는 다음 항목을 직접 제어할 수 있어야 한다.

- 로그인 세션/쿠키 정리
- 동기화 결과에서 가져올 항목 선택
- 가져오지 않을 항목 복구
- 삭제된 task 복구

## 8. 보안 리스크와 대응

| 리스크 | 대응 |
| --- | --- |
| 계정 정보 저장 우려 | 아이디/비밀번호 미저장, WebView 로그인 사용 |
| 세션 탈취 우려 | 세션/쿠키 정리 기능 제공, 세션 값 로그 출력 금지 |
| 과도한 e-campus 접근 | 수동 동기화 중심 |
| 민감정보 로그 노출 | 계정, 쿠키, 세션 로그 출력 금지 |
| HTML 구조 변경 | 파싱 실패 처리와 오류 안내 |
