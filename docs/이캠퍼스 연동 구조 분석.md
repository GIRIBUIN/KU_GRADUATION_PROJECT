# e-campus 연동 구조 분석

## 1. 연동 목적

e-campus 체크리스트에 있는 온라인 강의, 과제, 팀프로젝트 과제 등의 학업 ToDo를 가져와 앱의 task로 관리하기 위해 연동한다.

## 2. 기본 원칙

- e-campus 데이터는 읽기 전용으로 수집한다.
- 사용자의 제출, 수정, 삭제를 e-campus에 반영하지 않는다.
- 앱은 e-campus 원본 시스템을 변경하지 않는다.
- 동기화는 필요한 순간에만 수행한다.

## 3. 로그인 방식

### 현재 구현

```text
동기화 요청
→ 로그인 세션이 없으면 WebView 로그인
→ WebView에서 e-campus 로그인 완료 감지
→ 쿠키를 EcampusSession으로 수집
→ e-campus 체크리스트 수집
```

현재 구현은 아이디와 비밀번호를 저장하지 않는다. 로그인 세션은 메모리 상태와 WebView 쿠키를 통해 유지되며, 설정 화면에서 세션/쿠키 정리를 실행할 수 있다.

## 4. 자동 동기화

- 기본 동기화는 수동 동기화이다.
- 현재 UI와 실행 흐름에는 앱 실행 시 자동 동기화가 구현되어 있지 않다.
- `autoSyncEnabled`, `saveEcampusAccount` 설정 필드는 남아 있지만 현재 화면에서는 사용하지 않는다.

## 5. 동기화 결과 분류

### 가져올 항목

- DB에 없는 신규 e-campus 항목
- `sourceKey`는 같지만 원본 정보가 달라진 업데이트 후보

### 가져오지 않을 항목

- 이미 가져온 항목
- 완료된 항목
- 삭제된 항목
- 제외된 항목

## 6. 업데이트 후보

`sourceKey`가 같고 다음 원본 정보가 달라진 경우 업데이트 후보로 본다.

- `sourceTitle`
- `sourceDueDate`
- `sourceCourse`
- `sourceType`

업데이트 후보는 사용자가 선택했을 때만 task 값을 원본 기준으로 덮어쓴다.

## 7. 원본에서 사라진 항목

현재 구현은 다음 조건을 모두 만족하는 e-campus task를 완료 처리한다.

- 기존 task 상태가 `active`
- `sourceKey`가 있음
- 이번 동기화 결과에 같은 `sourceKey`가 없음
- `sourceDueDate` 또는 `dueDate`가 동기화 시각 이후임

마감일이 지났거나 마감일이 없는 항목은 자동 완료하지 않고 기존 상태를 유지한다.

## 8. 동기화 실패 케이스

- 로그인 실패
- 비밀번호 변경
- 네트워크 오류
- e-campus 서버 오류
- HTML 구조 변경
- 세션 만료

실패 시 사용자에게 원인을 구분할 수 있는 메시지를 제공한다.

## 9. 보안 처리

- 현재 구현은 계정 아이디/비밀번호 저장을 제공하지 않는다.
- 자체 서버에 계정 정보를 저장하지 않는다.
- WebView 로그인 후 쿠키 기반 세션으로 체크리스트를 조회한다.
- 사용자는 설정 화면에서 세션과 쿠키를 정리할 수 있다.
- 로그에는 계정 정보와 세션 값을 남기지 않는다.
