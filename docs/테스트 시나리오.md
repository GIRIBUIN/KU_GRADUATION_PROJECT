# 테스트 시나리오

## 1. 개인 Task

### 개인 task 생성

1. 추가 버튼을 누른다.
2. 제목, 마감일, 태그, 우선순위, 알림을 입력한다.
3. 저장한다.
4. 홈과 목록에 task가 표시되는지 확인한다.

### 개인 task 수정

1. task를 선택한다.
2. 제목 또는 마감일을 수정한다.
3. 저장한다.
4. 목록에 수정 결과가 반영되는지 확인한다.

### 개인 task 삭제/복구

1. task를 삭제한다.
2. 목록에서 사라지는지 확인한다.
3. 관리 탭의 삭제된 작업에서 보이는지 확인한다.
4. 복구 시 다시 목록에 표시되는지 확인한다.

## 2. e-campus 동기화

### 신규 항목 가져오기

1. 동기화 버튼을 누른다.
2. e-campus 체크리스트를 수집한다.
3. 신규 항목이 `가져올 항목`에 표시되는지 확인한다.
4. 선택 항목 가져오기를 누른다.
5. 홈과 목록에 task가 추가되는지 확인한다.

### 가져오지 않을 항목

1. 동기화 결과에서 특정 항목을 가져오지 않는다.
2. 해당 항목이 `excluded` 상태의 e-campus task로 저장되는지 확인한다.
3. 다음 동기화 때 같은 항목이 신규가 아니라 `가져오지 않을 항목`에 표시되는지 확인한다.

### 삭제된 e-campus task

1. 가져온 e-campus task를 삭제한다.
2. 목록에서 사라지는지 확인한다.
3. 다음 동기화 때 같은 항목이 `가져오지 않을 항목`에 표시되는지 확인한다.

### 업데이트 후보

1. 같은 `sourceKey`를 가진 항목의 원본 마감일 또는 제목이 달라진 상황을 만든다.
2. 동기화 결과에서 업데이트 후보로 표시되는지 확인한다.
3. 사용자가 선택한 경우에만 기존 task 값이 원본 기준으로 갱신되는지 확인한다.

## 3. 완료 처리

1. task를 완료 처리한다.
2. 홈의 오늘 할 일 또는 마감 임박 목록에서 처리 방식이 맞는지 확인한다.
3. 완료 필터에서 표시되는지 확인한다.
4. e-campus에서 원본 항목이 사라진 active task 중 마감 전 항목은 동기화 후 완료 처리되는지 확인한다.
5. 마감일이 지났거나 sourceKey가 없는 e-campus task는 원본 목록에 없어도 자동 완료되지 않는지 확인한다.

## 4. Parent/Sub Task

1. parent task에 sub task를 추가한다.
2. 홈에서 sub task 목록 대신 진행률이 표시되는지 확인한다.
3. 목록에서 parent task를 펼쳐 sub task가 보이는지 확인한다.
4. 상세 화면에서 sub task 완료 상태를 변경할 수 있는지 확인한다.
5. sub task 전체 완료가 parent task 자동 완료로 이어지지 않는지 확인한다.

## 5. 태그/우선순위

1. 태그를 생성한다.
2. 태그 색상을 선택하거나 직접 입력한다.
3. task에 태그를 연결한다.
4. 태그를 숨김 처리하면 관리 화면과 홈 태그 선택에서 표시 정책이 맞는지 확인한다.
5. 태그를 특정 폴더에 배치하면 관리 화면에서 폴더 그룹에 표시되는지 확인한다.
6. task 우선순위 정렬은 task에 직접 저장된 우선순위만 사용하는지 확인한다.

참고: 현재 코드에는 `Tag` 모델의 태그 기본 우선순위 필드가 없으므로 태그 기반 우선순위 테스트는 수행하지 않는다.

## 6. 폴더

1. 폴더를 생성한다.
2. 상위 폴더를 지정해 중첩 폴더로 저장되는지 확인한다.
3. 폴더 숨김 처리 시 관리 화면 표시 정책이 맞는지 확인한다.
4. task 생성/수정 화면에서 폴더를 연결하거나 제거한다.
5. task가 여러 폴더에 들어갈 수 있는지 확인한다.
6. e-campus 동기화 시 `e-campus` 폴더가 자동 생성/연결되는지 확인한다.

## 7. 알림

1. task에 알림을 설정한다.
2. 마감일 기준 N분 전 특정 시간으로 예약되는지 확인한다. (현재 코드에서 `daysBeforeDue`는 분 단위로 해석됩니다)
3. 알림을 끄면 예약이 해제되는지 확인한다.
4. task 마감일이 변경되면 알림 시간이 갱신되는지 확인한다.

## 8. 설정/보안

1. e-campus 동기화를 실행한다.
2. 로그인 세션이 없으면 WebView 로그인 화면으로 이동하는지 확인한다.
3. 로그인 완료 후 쿠키 기반 세션으로 todo를 가져오는지 확인한다.
4. 설정 화면에서 로그인 세션 상태와 마지막 동기화 시간이 표시되는지 확인한다.
5. 세션/쿠키 정리 실행 후 세션 상태가 없어지는지 확인한다.
6. 아이디와 비밀번호가 앱 설정 또는 DB에 저장되지 않는지 확인한다.

참고: 현재 UI에는 계정 정보 저장 및 앱 실행 시 자동 동기화 기능이 없으므로 해당 테스트는 제외한다.

## 9. 실패 케이스

- 로그인 실패
- 네트워크 오류
- e-campus 응답 오류
- HTML 구조 변경
- sourceKey 생성 실패
- 마감일 파싱 실패

각 실패 케이스에서 앱이 중단되지 않고 사용자에게 적절한 안내를 제공하는지 확인한다.
