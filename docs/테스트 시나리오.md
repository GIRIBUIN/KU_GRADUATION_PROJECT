# 테스트 시나리오

## 1. 개인 Task

### 개인 task 생성

1. 추가 버튼을 누른다.
2. 제목, 마감일, 태그, 우선순위, 알림을 입력한다.
3. 저장한다.
4. 홈과 목록에 task가 표시되는지 확인한다.

### 개인 task 수정

1. task를 선택한다.
2. 제목 또는 마감일을 수정한다.
3. 저장한다.
4. 목록에 수정 결과가 반영되는지 확인한다.

### 개인 task 삭제/복구

1. task를 삭제한다.
2. 목록에서 사라지는지 확인한다.
3. 관리 탭의 삭제된 작업에서 보이는지 확인한다.
4. 복구 시 다시 목록에 표시되는지 확인한다.

## 2. e-campus 동기화

### 신규 항목 가져오기

1. 동기화 버튼을 누른다.
2. e-campus 체크리스트를 수집한다.
3. 신규 항목이 `가져올 항목`에 표시되는지 확인한다.
4. 선택 항목 가져오기를 누른다.
5. 홈과 목록에 task가 추가되는지 확인한다.

### 가져오지 않을 항목

1. 동기화 결과에서 특정 항목을 가져오지 않는다.
2. 다음 동기화 때 같은 항목이 신규가 아니라 `가져오지 않을 항목`에 표시되는지 확인한다.

### 삭제된 e-campus task

1. 가져온 e-campus task를 삭제한다.
2. 목록에서 사라지는지 확인한다.
3. 다음 동기화 때 같은 항목이 `가져오지 않을 항목`에 표시되는지 확인한다.

### 업데이트 후보

1. 같은 `sourceId/value`를 가진 항목의 원본 마감일 또는 제목이 달라진 상황을 만든다.
2. 동기화 결과에서 업데이트 후보로 표시되는지 확인한다.
3. 사용자가 선택한 경우에만 기존 task 값이 원본 기준으로 갱신되는지 확인한다.

## 3. 완료 처리

1. task를 완료 처리한다.
2. 홈의 오늘 할 일 또는 마감 임박 목록에서 처리 방식이 맞는지 확인한다.
3. 완료 필터에서 표시되는지 확인한다.
4. e-campus에서 원본 항목이 사라져도 앱 task가 자동 완료되지 않는지 확인한다.

## 4. Parent/Sub Task

1. parent task에 sub task를 추가한다.
2. 홈에서 sub task 목록 대신 진행률이 표시되는지 확인한다.
3. 목록에서 parent task를 펼쳐 sub task가 보이는지 확인한다.
4. 상세 화면에서 sub task 완료 상태를 변경할 수 있는지 확인한다.
5. sub task 전체 완료가 parent task 자동 완료로 이어지지 않는지 확인한다.

## 5. 태그/우선순위

1. 태그를 생성한다.
2. 태그 기본 우선순위를 설정한다.
3. task에 태그를 연결한다.
4. task 우선순위가 없을 때 태그 기본 우선순위가 적용되는지 확인한다.
5. 태그가 여러 개인 경우 가장 높은 태그 우선순위가 적용되는지 확인한다.

## 6. 폴더

1. 폴더를 생성한다.
2. task를 폴더에 직접 추가한다.
3. 태그별 task 목록에서 선택 추가 또는 일괄 추가를 수행한다.
4. 폴더에서 task를 제거한다.
5. task가 여러 폴더에 들어갈 수 있는지 확인한다.

## 7. 알림

1. task에 알림을 설정한다.
2. 마감일 기준 N일 전 특정 시간으로 예약되는지 확인한다.
3. 알림을 끄면 예약이 해제되는지 확인한다.
4. task 마감일이 변경되면 알림 시간이 갱신되는지 확인한다.

## 8. 설정/보안

1. 계정 정보 저장에 동의한다.
2. 자동 동기화를 켠다.
3. 앱 실행 시 1회 동기화가 시도되는지 확인한다.
4. 저장된 계정 삭제 후 자동 동기화가 실행되지 않는지 확인한다.
5. 동기화 후 세션/쿠키 정리가 수행되는지 확인한다.

## 9. 실패 케이스

- 로그인 실패
- 네트워크 오류
- e-campus 응답 오류
- HTML 구조 변경
- sourceId 누락
- 마감일 파싱 실패

각 실패 케이스에서 앱이 중단되지 않고 사용자에게 적절한 안내를 제공하는지 확인한다.
